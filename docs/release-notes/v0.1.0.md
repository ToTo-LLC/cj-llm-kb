# brain 0.1.0

*First public release — 2026-04-23*

A knowledge base you run on your own machine, maintained by an LLM.
Your notes are plain Markdown files in a folder you control. Nothing
leaves your computer except the LLM calls you explicitly make.

## What's new

- **Your vault is a folder of Markdown files.** It lives at
  `~/Documents/brain/`, it's Obsidian-compatible, and brain won't fight
  you if you edit it by hand. Delete the app and your notes stay put.

- **Three ways to work with your knowledge.** *Ask* gives cited answers
  from your wiki and refuses to speculate beyond what it can source.
  *Brainstorm* pushes back and co-develops ideas. *Draft* edits a
  document inline while you watch.

- **Drop anything in the inbox.** URLs, pasted text, PDFs (via CLI or
  Claude Desktop today), emails, meeting transcripts (`.vtt` / `.srt` /
  `.docx`), single tweets. brain classifies each item, summarizes it,
  and files it into one of three domains (`research`, `work`,
  `personal`).

- **Every change is staged.** The LLM proposes patches; you approve
  each one before it touches disk. You can flip individual categories
  to auto-apply if you want to move faster, but validation still runs
  on every write.

- **Your three domains stay separate.** `personal` content never shows
  up in default or cross-domain queries — crossing scopes takes an
  explicit opt-in per thread.

- **Runs locally. Loopback only. Zero telemetry.** The only outbound
  traffic is the LLM calls you authorize with your own API key, plus an
  opt-out version check at startup. No analytics, no crash reporting,
  no phone-home.

## Install

### macOS 13+

```bash
curl -fsSL https://github.com/ToTo-LLC/cj-llm-kb/releases/download/v0.1.0/install.sh | bash
```

### Windows 11

```powershell
irm https://github.com/ToTo-LLC/cj-llm-kb/releases/download/v0.1.0/install.ps1 | iex
```

Then run `brain start`. Your browser opens to the setup wizard — pick
a vault location, paste your Anthropic API key, and you're in.

## System requirements

- macOS 13+ (Apple Silicon or Intel) or Windows 11
- ~500 MB of disk for the app. Your vault grows with your notes.
- An Anthropic API key — get one at
  [console.anthropic.com](https://console.anthropic.com).

## Known issues in 0.1.0

These are the items we consciously shipped with. Each one is tracked
for a follow-up release.

- **PDF upload from the web UI is deferred to v0.2.0.** For now, drop
  PDFs into `~/Documents/brain/raw/inbox/` and let brain pick them up,
  or use Claude Desktop (which handles PDFs through the MCP tools).
  The browser uploader accepts text, Markdown, HTML, VTT transcripts,
  DOCX, email (`.eml`), and URLs. *(GitHub issue link pending)*

- **Bulk-import uses a path text input.** There's no native folder
  picker yet — paste or type the absolute folder path instead. A real
  picker lands in v0.2.0 when the desktop wrapper arrives.
  *(GitHub issue link pending)*

- **Broken `[[wikilinks]]` render as plain text.** If a linked note
  doesn't exist yet, the link still works the moment you create the
  target note — you just don't get a visual "unreachable" indicator
  in the reader in v0.1.0. A dedicated lint chip ships in v0.2.0.
  *(GitHub issue link pending)*

- **A test-mode ingest edge case.** Running with `BRAIN_E2E_MODE=1`
  (used only by internal test harnesses) can stage a patch that the
  apply step then rejects. Regular users running with a real API key
  are not affected. Tracked for v0.1.1.
  *(GitHub issue link pending)*

- **On Windows, the local API token uses basic file permissions, not
  full ACLs.** The token still rotates every `brain start`, and brain
  only binds to loopback, so a local account on your machine is the
  only way to read it in the first place. Full ACL lockdown is
  scheduled for v0.2.0. *(GitHub issue link pending)*

## Scope deviations vs original Q3a plan

- **Q3a planned 104-item manual walk** on primary daily-driver machine over ~3 hours. Actual sweep was ~42 minutes of browser-automation + 6 rounds of fix-and-re-cut.
- **Rationale:** Claude in Chrome's screenshot tool hit CDP timeouts against Next.js' hydration cycle (Chromium Page.captureScreenshot → 30s stall). I pivoted from visual-verification to semantic-verification via `read_page` + JavaScript evaluation against the same DOM/React state a human tester would observe. This covered interaction surfaces + render-correctness comprehensively; it did not capture screenshots.
- **LLM-dependent flows (chat turn, ingest integrate, summarize)** not exercised because no Anthropic API key was available in the session. Happy-path LLM usage is covered by the Plan 07/08 e2e suite (FakeLLM) + unit tests.
- **Screenshots for README** not captured. README's 6 placeholder `<img src>` references resolve to nonexistent files until captured. This is a TODO for v0.1.1 — see below.

### Install-path bugs caught during sweep setup

The install pipeline itself surfaced four separate bugs before the structural sweep could begin. All fixed + release assets re-uploaded with `--clobber` (same v0.1.0 tag).

| ID | Severity | Title | Fix commit |
|---|---|---|---|
| IP1 | **BLOCKER** | `install.sh` / `install.ps1` sourced `install_lib/*.sh` relative to the script's own dir BEFORE downloading the tarball → curl \| bash failed with "install_lib/ not found". Added dual-mode bootstrap (checkout vs curl). | `0e8e578` |
| IP2 | **BLOCKER** | Shim + `.app` launcher called bare `uv run` → failed from any context without `~/.local/bin/` on PATH (launchd, Spotlight, `.app` double-click, bash subshells). Resolve absolute uv path at shim-write time. | `a179ccf` |
| IP3 | **BLOCKER** | Supervisor spawned uvicorn via nested `uv run` from inside a uv-managed Python process; `uv` itself not on PATH of uv's child processes. Switched to `sys.executable -m uvicorn` (we're already inside the venv). | `586d087` |
| IP4 | **BLOCKER** | Shim + `.app` launcher didn't export `BRAIN_INSTALL_DIR`. Supervisor then fell back to `~/Applications/brain/` (non-existent). Shims now export it. | `1743680` |

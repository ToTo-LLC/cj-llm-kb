# v0.1.0 install walkthrough (clean VM + primary machine)

Used for Plan 09 Tasks 9, 10, 11. The repo is public and v0.1.0 is live
on GitHub, so the install is now a single anonymous curl — no host-side
local tarball server needed.

Release URL: https://github.com/ToTo-LLC/cj-llm-kb/releases/tag/v0.1.0

## At-a-glance flow (any of the 3 target environments)

1. Provision a clean environment (fresh macOS VM, fresh Windows 11 VM,
   or your primary machine — see env-specific sections below).
2. Run the one-command install.
3. Walk `brain doctor`, `brain start`, the setup wizard, one ingest,
   `brain uninstall`.
4. Fill in the matching receipt template under `docs/testing/`.

---

## Task 9 — Clean-Mac VM dry run

### 9.1 Provision

```bash
# On the host Mac, one-time:
brew install cirruslabs/cli/tart
tart pull ghcr.io/cirruslabs/macos-sonoma-base:latest

# Each run:
tart clone ghcr.io/cirruslabs/macos-sonoma-base:latest brain-mac-vm
tart run brain-mac-vm   # opens a VM window; complete first-boot setup once
```

Any of Tart / UTM / Parallels works — Tart is fastest on Apple Silicon.
First boot asks for a local user account; remember the password.

### 9.2 Inside the VM

Open **Terminal** (⌘+space → "Terminal"), then:

```bash
mkdir -p ~/brain-dryrun && cd ~/brain-dryrun

# Install (~30s on a warm network)
curl -fsSL https://github.com/ToTo-LLC/cj-llm-kb/releases/download/v0.1.0/install.sh \
  -o install.sh
bash install.sh 2>&1 | tee install.log

# Diagnostic
brain doctor 2>&1 | tee doctor.log

# Start brain — browser auto-opens at http://localhost:4317/
brain start 2>&1 | tee start.log

# Walk the setup wizard in the browser. Use a throwaway Anthropic API key
# for the wizard's API-key step (or copy yours from the host; either works).
#
# Then in the chat, type something like:
#   "hello"
# and approve the resulting pending patch from /pending.
#
# Once done:
brain stop 2>&1 | tee stop.log
brain uninstall --yes 2>&1 | tee uninstall.log
```

### 9.3 Screenshots to capture

Take 4 at natural moments (⌘+shift+4 selects an area):

1. `install-complete.png` — `install.log` ending with the success line.
2. `doctor-green.png` — `brain doctor` output with all PASS.
3. `setup-wizard.png` — the setup wizard in the browser.
4. `first-chat-response.png` — the first assistant response with the
   inline patch card.

Drop them into `docs/testing/screenshots/v0.1.0/mac/` on the host
(transfer via shared folder or AirDrop).

### 9.4 Receipt

Fill in `docs/testing/clean-mac-vm-receipt.md` using the Plan 08
template. At the top set:

- Release: `v0.1.0`
- Install URL: `https://github.com/ToTo-LLC/cj-llm-kb/releases/download/v0.1.0/install.sh`
- SHA256: `edf81cbb921437f55c6415402bcf19adcdc4548dfb15a4cfbb05c6c09d72a7c5`

Paste each log from the commands above into the matching step section
and embed the 4 screenshots.

Status line at the end: `✓ PASS` or `⚠ PASS WITH NOTES` or `✗ FAIL`.

## Task 10 — Clean-Windows VM dry run

### 10.1 Provision

```bash
# On the host Mac, one-time:
brew install --cask utm
# Download Windows 11 ARM ISO + install into UTM (~15 GB).
# Or use Parallels (licensed).
```

Snapshot the clean post-install state before each dry run so you can
roll back quickly.

### 10.2 Inside the VM

Open **PowerShell** (Start → "PowerShell" — Windows PowerShell 5.1 is
fine; PowerShell 7 also works):

```powershell
New-Item -ItemType Directory -Force ~/brain-dryrun | Out-Null
Set-Location ~/brain-dryrun

# Install
iwr https://github.com/ToTo-LLC/cj-llm-kb/releases/download/v0.1.0/install.ps1 `
    -OutFile install.ps1
powershell.exe -ExecutionPolicy Bypass -File .\install.ps1 *>&1 | Tee-Object install.log

# Diagnostic
brain doctor *>&1 | Tee-Object doctor.log

# Start — browser auto-opens
brain start *>&1 | Tee-Object start.log

# Walk wizard, send one chat message, approve one pending patch.

brain stop *>&1 | Tee-Object stop.log
brain uninstall --yes *>&1 | Tee-Object uninstall.log
```

### 10.3 Screenshots

Same 4 as Task 9 (PowerShell window, browser). Drop into
`docs/testing/screenshots/v0.1.0/windows/`.

### 10.4 Receipt

Fill in `docs/testing/clean-windows-vm-receipt.md`. Same metadata as
Task 9 except Install URL ends in `install.ps1`.

## Task 11 — Full 104-item manual-QA sweep (primary machine)

### 11.1 Scope

Q3a decision: walk every item in
[`docs/testing/manual-qa.md`](manual-qa.md) on your primary daily-driver
machine, installed from the v0.1.0 release. Target: ~3 hours.

### 11.2 Install on primary machine

If you already have a dev install, stop it first:

```bash
brain stop || true
```

Install v0.1.0 into a separate dir from the dev checkout:

```bash
BRAIN_INSTALL_DIR=~/Applications/brain-v0.1.0 \
BRAIN_VAULT_ROOT=~/Documents/brain-v0.1.0-qa \
  curl -fsSL https://github.com/ToTo-LLC/cj-llm-kb/releases/download/v0.1.0/install.sh \
  | bash
```

The override of `BRAIN_VAULT_ROOT` keeps your real vault untouched.
Remove the QA install + vault with `brain uninstall --delete-vault` when
done.

### 11.3 Walk the checklist

- Work through every section of `docs/testing/manual-qa.md` (≈104
  items). Mark each `✓ PASS`, `⚠ PASS WITH NOTES`, or `✗ FAIL`.
- Capture **6–8 screenshots** at natural moments (for README — Task 12):
  `setup-wizard-step-3.png`, `new-thread-empty.png`,
  `pending-patch-diff.png`, `browse-backlinks.png`,
  `settings-domains.png`, `bulk-dry-run.png`, `inbox-drag-drop.png`,
  `chat-first-response.png`. Drop them into
  `docs/testing/screenshots/v0.1.0/`.

### 11.4 Receipt

Create `docs/testing/v0.1.0-qa-receipt.md` with:

- Metadata block: date, machine (model + OS version), release version,
  vault path, Anthropic model used.
- Per-section status + findings.
- Any FAIL items — fix inline (commit with `fix(...)` prefix) or move to
  `docs/v0.1.0-known-issues.md` as RELEASE_NOTE. Re-walk any fixed
  items.
- Overall status line.

Hard gate before Task 13: **overall receipt status `✓ PASS`** (with all
FAILs either resolved or triaged).
